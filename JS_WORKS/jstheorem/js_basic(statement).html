타입 변환과 단축 평가


  // 나중에 다시 찾아 볼 것. 지금은 딱히 읽어도 어디에 적용해야할지 모르겠음 ㅋ.ㅋ
  1. 타입 변환 -> 은 나중에....귀찮...
  
  2. 단축 평가 -> if 문을 대체할 수 있다.
    논리 연산의 결과를 결정하는 피연산자 타입을 변환하지 않고 그대로 반환을 한다. 
    다음과 같은 규칙을 따른다. 

    * 단축 평가 표현식 | * 평가 결과  
    true || anything | true
    false || anything | anything
    true && anything | anything
    false && anything | false

    - 논리곱&& 연산자 : 좌항에서 우항으로 평가 진행됨. 
      두 개의 피연산자가 모두 true일 때 true를 반환함.

    // [예제1] &&
    'Cat' && 'Dog' // -> 'Dog'
    논리곱 연산자는 좌항 우항 모두 true여야한다. 
    좌항이 true로 정해져도 우항까지 봐야하며, 이말은 즉 논리 연산의 결과를 결정하는것은 우항이라는 뜻.
    그래서 결과가 Dog가 출력된다.

    // [예제2] ||
    'Cat' || 'Dog' // -> 'Cat'
    두 개의 피연산자 중 하나만 true로 평가되어도 true를 반환한다.
    여기서는 논리 연산의 결과를 결정한 첫 번째 연산자를 그대로 반환한다.

    // 논리합 ||
    'Cat' || 'Dog' // -> 'Cat'
      // or은 일단 좌항부터 평가되고 true가 하나라도 있으면 true 이기에 cat 출력
    false || 'Dog' // -> 'Dog'
      // false 이더라도 or연산자에서는 true가 있다면 ture값을 결정하는 Dog가 출력
    'Cat' || false // -> 'Cat'
      // 위와 같은 설명

    // 논리곱  &&
    'Cat' && 'Dog' // -> 'Dog'
    false && 'Dog' // -> 'false'
    'Cat' && false // -> 'false'

    // 일단 기본적인 논리 연산의 규칙을 따르고, 값을 반환한다는 거!

일반적으로 위 -> 아래로 순차적으로 실행되는 코드를 인위적으로 제어할 수 있다.


(forEach, map, filter, reduce와 같은 고차 함수를 사용하는 함수형 프로그래밍을 사용하여 복잡성을 해결하려 노력한다.)

5-1. 블록문


  // 블록문 : {} 로 묶은 것. 코드 블록이라 한다. 하나의 실행 단위이다. 블록문은 언제나 문의 종료의 의미를 내포하므로 굳이 ; 붙이지 않는다.

조건문


  주어진 조건식의 평가 결과에 따라 코드 블록의 실행을 결정한다.
  조건식은 불리언 값으로 평가되는 표현식이다.

조건문) if...else


  조건식(불리언으로 평가되는 표현식)의 평가 결과, 
  즉 '논리적 참 거짓'에 따라 실해할 코드 블록을 결정한다.

  // if...else 문 
  if( 조건식 ) {
    // 조건식이 참이면 이 코드 블록을 실행한다.
  } else {
    // 조건식이 거짓이면 이 코드 블록이 실행된다.
  }

  // if문의 조건식은 불리언 값으로 평가되어야 한다. 
  // 조건식을 추가하여 조건에 따라 실행될 코드 블록을 늘릴 수 있다.

  // else if 문 사용
  if( 조건식1 ) {
    // 조건식1이 참이면 이 코드 블록을 실행합니다.
  } else if ( 조건식2 ) {
    // 조건식2가 참이면 이 코드 블록을 실행합니다.
  } else {
    // 조건식1과 조건식2 모두 거짓일 경우에 이 블록을 실행합니다.
  }

  // else if문과 else문은 옵션이다. 

  


  // [예제]
  var num = 2;
  var kind;

  // if 문
  if (num > 0) {
    kind = '양수'; // -> 음수를 구별할 수 없다.
  }
  console.log(kind); // 양수

  // if...else 문 
  if (num > 0) {
    kind = '양수';
  } else {
    kind = '음수'; // -> 음수를 구별할 수 있으나 0은 음수가 아님.
  }
  console.log(kind);

  // if...else if 문
  if (num > 0) {
    kind = '양수';
  } else if (num < 0) {
    kind = '음수';
  } else {
    kind = '영';
  }
  console.log(kind); // 양수

  👉 위 코드와 같이 만약 코드 블록 내의 문이 하나뿐이라면 중괄호를 생략할 수 있다.
  if (num > 0) kind = '양수';
  else if (num < 0) kind = '음수';
  else kind = '영';

조건문) switch


  주어진 표현식을 평가한다. 그 값과 일치하는 표현식을 갖는 case문으로 실행 흐름을 옮긴다.
  switch문의 표현식과 일치하는 case문이 없다면 실행 순서는 default 문으로 이동한다.
  (default는 선택사항으로 사용할 수도 있고, 사용하지 않을 수도 있다.)

  다양한 상황(case)에 따라 실행할 코드 블록을 결정한다.

  // swtich 문
  
  switch (표현식) {
    case 표현식1 : switch 문의 표현식과 표현식1이 일치하면 실행될 문;
      break;
    case 표현식2 : switch 문의 표현식과 표현식2가 일치하면 실행될 문;
      break; // 👉 case 다음에는 break문을 써주어 코드 블록에서 탈출 시켜준다.
    default : switch 문의 표현식과 일치하는 case문이 없을 때 실행될 문;
    // 👉 default 문의 실행이 종료되면 switch문을 빠져나간다. (break문을 생략하는 것이 일반적이다.)
  }


  👉 '폴 스루(fall through)'
  break문을 쓰지 않을 경우, 
  switch문의 표현식과 case 표현식(n)이 일치하는 실행문을 돌리고, 그 다음 case들도 실행하게 된다.

  // 폴 스루를 이용한 윤년 계산
  var year = 2000;
  var month = 2;
  var days = 0;
  
  switch (month) {
    case 1: case 3: case 5: case 7: case 8: case 10: case 12: 
      days = 31;
      break;
    case 4: case 6: case 9: case 11:  
      days = 30;
      break;
    case 2: 
      days = ((year % 4 === 0 && year % 100 !== 0) || (year % 400 === 0)) ? 29 : 28;
      break;
    default:
      console.log('Invalid month');
  }
  console.log(days) // 29

반복문

반복문은 조건식의 평과 결과가 '참'인 경우 코드 블록을 실행한다.


그 후 조건식을 다시 평가하여 여전히 '참'인 경우 코드 블록을 다시 실행한다.


조건식이 거짓일 때 까지 반복한다.


JS에서 반복문은 for, while, do...while이 있다.


  - for : 반복 범위가 명확할 때 사용.
  - while : 반복 횟수가 불명확할 때 주로 사용. 조건문이 거짓일 경우에는 코드 블록을 실행하지 않고 종료.
  - do...while : 먼저 코드 블록을 한 번 실행하고 조건식을 평가.

반복문) for


  // [for문의 형태]
  for (변수 선언문 또는 할당문; 조건식; 증감식) {
    조건식이 참일 경우 반복 실행될 문;
  }

  // [무한루프]
  for(;;) { ... }

  for문의 동작 순서
  1. 변수 선언문 또는 할당문은 '단 한 번만' 실행된다.
  2. '조건식을 평가'한다. 'true'일 경우...
  3. 조건식이 참일 경우 '{코드 블록}' 안에 있는 반복 실행될 문을 실행한다.
  4. 코드 블록의 실행이 종료되면 '증감식'이 실행된다.
  5. 다시 2. 조건식을 평가한다. 
  6. 다시 3. {코드 블록}을 실행하고
  7. 다시 4. 증감식을 실행한다.
  ... 조건식의 평가 결과가 false일 경우 for문의 실행이 종료된다.

반복문) while


  주어진 조건식의 평가 결과가 참이면 코드 블록을 계속 반복 실행한다.

  // [예제1 - count가 3보다 작을 때까지 코드 블록을 계속 반복 실행한다.] 
  var count = 0;
  while (count < 3) {
    console.log(count); // 0 1 2
    count++;
  }

  // 조건식의 결과가 언제나 참이면 [무한루프]
  while (true) { ... }

  // [예제2 - 무한루프에서 탈출하기 위해서는 코드 블록 내에 if문으로 탈출 조건을 만들고 break문으로 코드 블록을 탈출한다.]
  var count = 0;
  while (true) {
    console.log(count);
    count++;
    // count가 3이면 코드 블록을 탈출한다.
    if (count === 3) break;
  }

반복문) do...while


  코드 블록을 먼저 실행하고 조건식을 평가한다.
  코드 블록은 한 번 이상은 무조건 실행된다.

  // [예제1]
  var count = 0;

  do {
    console.log(count); // 0 1 2
    count++;
  } while (count < 3);

break 문


  break문은 코드 블록을 탈출하는데, 정확히 말하면
  레이블문, 반복문, swtich문의 코드 블록을 탈출한다.

  // [문자열에서 특정 문자의 인덱스(위치)를 검색]

  var string = 'Hello World!';
  var search = 'l';
  var index;

  // 문자열은 유사배열이므로 for문으로 순회할 수 있다.
  for(var i = 0; i < string.length; i++) {
    // 문자열의 개별 문자가 'l'이면
    if (string[i] === search) {
      index = i;
      break;
    }
  }
  console.log(index); // 2

  // 참고로 String.prototype.indexOf 메서드를 사용해도 같은 동작을 한다.
  console.log(string.indexOf(search)); // 2

continue 문


  continue는 반복문의 코드 블록 실행을 '현 시점에서 중단'하고,
  반복문의 '증감식'으로 실행 흐름을 이동시킨다.
  break문처럼 '반복문을 탈출하지는 않는다.'

  // [continue를 이용한 문자열에서 특정 문자열의 개수 검색]

  var string = 'Hello World!';
  var search = 'l';
  var count = 0;
  
  // 문자열은 유사 배열이므로 for문으로 순회할 수 있다.
  for (var i = 0; i < string.length; i++) {
    // 'l'이 아니면 현 지점에서 실행을 중단하고 반복문의 증감식으로 이동한다.
    if (string[i] !== search) continue;
    count++;
  }
  console.log(count); // 3
  
  // String.prototype.match 메서드를 사용해도 같은 동작을 한다.
  const regexp = new RegExp(search, 'g');
  console.log(string.match(regexp).length); // 3



  // continue문을 사용하면 훨씬 간편하고 가독성이 좋다.

  // 1) continue를 쓰지 않으면 if문 내에서 코드를 작성해야 한다.
  for (let i = 0; i < string.length; i++) {
    
    // 'l'이면 카운트를 증가시킨다.
    if (string[i] === search) {
      count++;
      // 그리고 또 여기서
      // 추가적으로 행위를 
      // 할 듯...
    }
  }

  // 2) continue를 쓰면 if문 밖에서 코드를 작성할 수 있다!
  for (let i = 0; i < string.length; i++) {
    
    // 'l'이 아니면 카운트를 증가시키지 않고 증감식으로 돌아간다.
    if (string[i] !== search) continue;
    
    count++;
    // if문 밖에서 
    // 추가적인 코드를
    // 넣으니 가독성이 좋다.
  }
